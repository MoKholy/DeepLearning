Music Genre Classification

  • Mohammad ElKholy
  • Allaa ElKhouly

Problem Statement

Investigating the use of Audio Spectrogram Transformers for Music Genre Classification.

Dataset

While GTZAN is a very popular Music Genre Classification dataset, we opted out of using it as Transformers are generally large models and require large amounts of data to train. Thus, we used the Free Music Archive dataset, which is a much larger dataset. We used the Free Music Archive-Small version of the Dataset. This dataset consists of 8000 entries of 8 of the most popular genres from the Free Music Archive-Medium version of the dataset. There are 1000 30s clips per genre, thus the dataset is balanced, with a total size of 7.4 GBs. The genres are:

  • Electronic
  • Experimental
  • Folk
  • Hip-Hop
  • Instrumental
  • International
  • Pop
  • Rock

Data PreProcessing

The dataset is provided in the form of mp3 files. We first convert the mp3 files to wav files as mp3 files cause errors when trying to load on the fly in the dataloader. We compute the log mel spectrogram on the fly with the same parameters as the original paper, we use a function provided by the transformer library from Hugging Face to do this. We normalize using the mean and standard deviation of the dataset, we calculated them by loading the dataset and computing the mean and standard deviation of the spectrograms. We use an 80/10/10 split for train/validation/test respectively.

Data Augmentation

Our original dataset was still small and the model did not converge initially, we thus used data augmentation to increase the size of the dataset. Since the clips were 30s long, we split each clip into 3 segments of 10 seconds each.
